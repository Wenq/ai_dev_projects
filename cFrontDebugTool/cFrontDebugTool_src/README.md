// 测试环境debug效率工具
# 该工具用于协助前端日常开发工作，减少手工重复动作，提高工作效率
## 功能包括但不限于：
支持浏览器： chrome/edge，内核88及以上版本（内外网都可用）
功能列表（基于当前打开的苍穹环境）：
1、一键进入debug模式
2、一键挂接本地前端开发环境做debug（前端本地环境地址可配置）
3、一键预览单据（pc和移动不同按钮），鼠标在任意页面选中单据Id后点击按钮进行预览
4、一键打开新旧开发平台
5、一键打开平台常用测试环境
6、一键在当前环境挂接其他环境前端代码
安装 - 使用chrome/edge的"管理扩展程序" - 启用"开发者模式" - "加载已解压的扩展程序"

# 测试环境debug效率工具 - 尝试用ai编程助手协作完成开发

## 支持范围
平台和业务的测试环境，包括星瀚、星空旗舰等（通过环境域名判断），不支持客户环境，因为会泄漏debug相关信息

    1. 新增chrome插件，提供功能面板或下拉菜单 -ok
    2. 支持点击菜单按钮一键进入debug模式 -ok
    3. 支持一键打开旧开发平台、新开发平台 -ok 
    4. 支持一键在一个当前测试环境挂另一个测试环境前端代码（比如sit挂dev-ok、后端本地环境挂前端开发本地环境-?、测试环境挂前端开发本地环境-ok）
    5. 一键打开某表单的设计器（可用户录入formId） -？【暂无方案】
    6. 可配置前端自己本地开发环境的代码，一键在当前测试环境挂接自己本地开发环境到当前测试环境 -ok
    7. 一键打开某单据（需要录入formId或鼠标在页面上选中）-ok
    8. 一键打开各测试环境：dev、sit、bos_baseline_a、baseline_a -ok
    9.一键打开移动端单据（url访问格式与pc不同）,默认带上当前页面上选中的formId -ok

## 新页签打开URl地址的拼接逻辑
    1. debug模式 - 保留完整url，如果kdweb_debug参数已存在则弹窗提示，不存在则新增该参数。
    2. 挂前端环境代码debug - 保留完整url，如果kdweb_cdn已存在则替换其值，不存在则新增该参数。
    3.挂上测试环境前端代码 - 同上。
    4.预览pc/mob单据 - 新建url，保留url里已有参数kdweb_debug、kdweb_cdn、kdcdc。
    5.打开开发平台/new开发平台 - 同上。
